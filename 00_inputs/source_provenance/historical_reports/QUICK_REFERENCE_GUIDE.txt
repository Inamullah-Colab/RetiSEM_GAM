WHAT WAS DONE - QUICK SUMMARY
================================================================================

Raw Data:
  • 9 XPT files from NHANES website
  • 10,348 total participants
  • 187 total variables

Processing Steps:
  1. Merged all 9 datasets on SEQN (Participant ID)
  2. Filtered to complete cases on key variables (n=6,606)
  3. Removed variables with >30% missing (87 removed, 100 kept)
  4. Imputed remaining missing values using median/mode
  5. Generated final clean dataset

Final Result:
  ✓ File: NHANES_final_causal.csv
  ✓ Size: 6,606 participants × 100 variables
  ✓ Quality: 0 missing values, ready for analysis

WHAT IMPUTATION WAS USED?
================================================================================

Rule: Variables with ≤30% missing got imputed

Method Used: MEDIAN IMPUTATION
  • For numeric variables: Filled with median value from available data
  • For categorical: Filled with mode (most common value)
  • Why: Simple, robust, preserves distribution, commonly used

Variables with Largest Imputations:
  DMQMILIT (military service):    1,810 values (27.4%)
  Sleep variables (SLQ030-240):   1,559 values each (23.6%)
  BMXSUB (subscapular skinfold):  1,203 values (18.2%)
  DMDMARTL (marital status):      1,093 values (16.5%)

Result: 0 missing values in final file ✓

Note: This is suitable because missing was <30%.
      For higher missingness, use MICE or multiple imputation.

================================================================================
METHODS YOU MIGHT APPLY
================================================================================

REGRESSION-BASED:
  • Linear regression (continuous outcomes)
  • Logistic regression (binary outcomes)
  • Poisson regression (count outcomes)
  ↓ Advantage: Simple, interpretable
  ↓ Disadvantage: Assumes correct functional form

PROPENSITY SCORE METHODS:
  • 1:1 Nearest neighbor matching
  • Caliper matching
  • Stratification on PS quintiles
  ↓ Advantage: Handles imbalance well
  ↓ Disadvantage: Loss of sample, reduced precision

INVERSE PROBABILITY WEIGHTING (IPTW):
  • Estimate propensity score
  • Weight by inverse probability
  • Fit marginal structural models
  ↓ Advantage: Uses full sample, simpler application
  ↓ Disadvantage: Extreme weights possible

DOUBLY ROBUST ESTIMATION:
  • Combine propensity score + outcome regression
  • Correct for either model misspecification
  ↓ Advantage: Robust to one model being wrong
  ↓ Disadvantage: More complex

G-COMPUTATION:
  • Standardization with outcome regression
  • Estimate potential outcomes for all
  ↓ Advantage: Causal contrasts directly
  ↓ Disadvantage: Parametric assumptions

IMPORTANT REMINDERS
================================================================================

✓ DO:
  • Document your assumptions (write them down!)
  • Pre-specify analysis plan (before seeing results)
  • Test sensitivity of findings (are they robust?)
  • Report effect sizes with confidence intervals
  • Check that final sample description makes sense
  • Consider survey design IF analyzing whole NHANES
  • Interpret results as associations first, causal if strong

✗ DON'T:
  • Treat p<0.05 as "causal proof" (it isn't!)
  • Ignore the 35% sample loss (document why)
  • Forget imputation was used (mention in methods)
  • Assume causality without strong theory
  • Do multiple testing without correction
  • Ignore confounding (use DAGs!)
  • Use extreme BP values without checking (data errors?)

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "I want to use variables from original 187"
Solution: Use NHANES_merged.csv instead of final file

Problem: "I want to apply different imputation method"
Solution: Use NHANES_cleaned_causal.csv (n=6,606 × 187, pre-imputation)
         Write your own imputation script

Problem: "Sample seems small compared to original"
Solution: This is expected! 
         Original 10,348 → Final 6,606 (63.8%)
         Loss due to: BP not measured on all, filtering rules
         Document this in your methods section

Problem: "Some BP values look impossible (e.g., 270)"
Solution: Likely data entry errors or measurement issues
         Check outliers: df[df['BPXSY1'] > 200]
         Consider excluding extreme values in sensitivity analysis

Problem: "Want to include variables that were removed"
Solution: Go back to NHANES_merged.csv
         Accept higher missingness (will need more imputation)
         Justify methodologically in paper
