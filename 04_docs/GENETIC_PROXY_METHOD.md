# Proxy-Genetic Covariate Method

This note documents how the `GREF_*` proxy-genetic variables were created for the RetiSEM-GAM release package.

## Purpose

Participant-level genotype data were not available in the active merged BC workflow. To avoid leaving ancestry-related structure entirely unmodeled, a demographic-to-reference proxy layer was created and used only as an adjustment block `G`.

These variables are:

- `GREF_AFR`
- `GREF_AMR`
- `GREF_EAS`
- `GREF_EUR`
- `GREF_SAS`
- `GREF_entropy`
- `GREF_major_superpop`
- `GREF_LABEL`

In the final mediation workflow, only:

- `GREF_AMR`
- `GREF_EUR`
- `GREF_SAS`
- `GREF_entropy`

were retained as the active `G` covariates.

## Construction Logic

The proxy variables were generated by the historical script:

- `00_inputs/source_provenance/historical_scripts/build_1000g_reference_proxy_features.py`

The script maps NHANES self-reported ethnicity code `RIDRETH1` to a fixed heuristic mixture over five 1000 Genomes superpopulations:

- AFR
- AMR
- EAS
- EUR
- SAS

The encoded mapping in the script is:

- `RIDRETH1 = 1` Mexican American
  - AFR 0.05, AMR 0.85, EAS 0.00, EUR 0.10, SAS 0.00
- `RIDRETH1 = 2` Other Hispanic
  - AFR 0.10, AMR 0.50, EAS 0.05, EUR 0.35, SAS 0.00
- `RIDRETH1 = 3` Non-Hispanic White
  - AFR 0.03, AMR 0.05, EAS 0.00, EUR 0.90, SAS 0.02
- `RIDRETH1 = 4` Non-Hispanic Black
  - AFR 0.90, AMR 0.02, EAS 0.00, EUR 0.08, SAS 0.00
- `RIDRETH1 = 5` Other/Multi
  - AFR 0.10, AMR 0.10, EAS 0.35, EUR 0.20, SAS 0.25

If `RIDRETH1` was unavailable or unmatched, the script assigned a weak uniform fallback:

- AFR 0.20
- AMR 0.20
- EAS 0.20
- EUR 0.20
- SAS 0.20

## Derived Variables

After assigning the five superpopulation weights, the script computes:

- `GREF_entropy`
  - Shannon-style entropy over the five weights, used as a summary of mixture spread
- `GREF_major_superpop`
  - the largest-weight superpopulation label
- `GREF_LABEL`
  - the original mapped NHANES category label

## Why These Are Covariates, Not Exposures

These variables are not individual-level genotypes, SNPs, PRS values, or ancestry estimates inferred from sequencing.

They are only demographic reference proxies. Therefore:

- they must not be interpreted as biological genetic effects
- they must not be promoted to primary causal exposures
- they are used only to reduce coarse population-structure confounding in sensitivity adjustment

This is why the final release locks `GREF_*` into the `G` covariate block rather than the exposure block.

## Suggested Paper Language

Recommended wording:

"Because participant-level genotype data were not available in the integrated real-data workflow, we included a proxy-genetic adjustment block derived from fixed heuristic mappings between NHANES ethnicity categories and 1000 Genomes superpopulation reference weights. These variables were used only as coarse ancestry-aware covariates and were not interpreted as participant genotypes or causal genetic exposures."

## Limitation

This proxy layer is a pragmatic adjustment device only. Any claim about true genetic mediation, inherited risk architecture, or population-specific causal mechanisms would require participant-level genotypes and a dedicated genetic analysis pipeline.
