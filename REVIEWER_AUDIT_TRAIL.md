# Reviewer Audit Trail

This file is the reviewer-facing map for checking the repository end to end.

It answers five questions directly:

1. What are the actual inputs?
2. Where did those inputs come from?
3. Which scripts generate the model-ready datasets?
4. Which scripts run the released models?
5. Which files contain the shipped results to compare against?

## 1. Actual inputs used in the released workflow

The active release workflow uses these main input tables:

- `00_inputs/raw_main/nhanes_core_for_merge.csv`
  - systemic NHANES-derived phenotype table
- `00_inputs/raw_main/macular_bc_prepared_with_seq.csv`
  - prepared retinal biomarker table used as the mediator source
- `00_inputs/raw_main/proxy_genetics_for_merge.csv`
  - proxy-genetic covariate table used only as adjustment block `G`

For quick inspection, lightweight reviewer samples are also included:

- `00_inputs/sample_small/`

## 2. Where those inputs came from

### NHANES clinical / phenotype inputs

The final workflow variables trace back to the raw NHANES XPT files copied under:

- `00_inputs/source_provenance/legacy_nhanes_xpt/`

Key examples:

- `DEMO_D.xpt`
  - age, sex, ethnicity, education, poverty ratio, weights
- `TRIGLY_D.xpt`
  - triglycerides including `LBXTR`
- `BIOPRO_D.xpt`
  - apolipoprotein B, insulin, creatinine
- `GLU_D.xpt`
  - glucose
- `GHB_D.xpt`
  - glycohemoglobin
- `CRP_D.xpt`
  - C-reactive protein
- `ALB_CR_D.xpt`
  - urine albumin and urine creatinine
- `BPX_D.xpt`
  - blood pressure outcomes

Use:

- `04_docs/DATASET_SOURCE_REFERENCE.md`
- `00_inputs/source_provenance/README.md`
- `04_docs/DATA_PROVENANCE_MAP.md`

Important:

- `00_inputs/source_provenance/historical_reports/` is archival provenance material
- it is not the active release definition
- if older reports differ from the current release docs, follow the current release docs

### Retinal mediator input

The active retinal source table is:

- `00_inputs/raw_main/macular_bc_prepared_with_seq.csv`

This repository treats that file as the prepared retinal feature source used by the active release branch.

The retinal features were extracted upstream from the APTOS diabetic retinopathy blindness dataset using the AutoMorph package, then matched into the participant-level workflow and further pruned in the released branch logic.

Use:

- `04_docs/DATASET_SOURCE_REFERENCE.md`
- `04_docs/RETINAL_SOURCE_PROVENANCE.md`
- `00_inputs/source_provenance/historical_scripts/merge_cvd_with_macular_full.py`
- `00_inputs/source_provenance/historical_reports/FULL_PROJECT_REPORT_START_TO_END.md`

Important limitation:

- the current release package does not rerun the raw APTOS image-to-feature extraction stage
- instead, it ships the prepared retinal feature table actually used in the released analysis
- the paper should be read alongside this repository for the broader retinal image-source and AutoMorph extraction narrative

So reviewers can fully inspect:

- the prepared retinal feature table actually used in the release
- how those retinal variables are matched by `SEQN`
- how those retinal variables are selected and transformed downstream

But reviewers cannot, from this release alone, rerun:

- retinal image acquisition
- raw fundus preprocessing
- original AutoMorph feature extraction on images

That distinction should be stated explicitly in the manuscript and repository.

## 3. How the model-ready branch datasets are generated

The active real-data builder scripts are:

- `01_scripts/real_data/01_build_global_only_branches.py`
- `01_scripts/real_data/04_build_exposure_outcome_cluster_pruned_branch.py`

Shared constants and transformations sit in:

- `01_scripts/real_data/_global_only_common.py`

The primary paper branch generated by these scripts is:

- `03_outputs/sensitivity_cluster_representative.csv`
- `03_outputs/sensitivity_cluster_representative_roles.json`

The stricter sensitivity branch generated by these scripts is:

- `03_outputs/sensitivity_exposure_outcome_cluster_pruned.csv`

and documented by:

- `04_docs/global_only_branch_manifest.json`

- `01_scripts/real_data/run_realdata_linear_nonlinear_compare.py`

- `RetiSEM_GAM`

- `01_scripts/real_data/06_run_exposure_outcome_cluster_pruned_comparison.py`

## 7. What can and cannot be checked from this repository alone

Can be checked directly:

- which final input tables are used
- which NHANES raw domains back the clinical variables
- how the two final branch datasets are built
- how `linear` and `RetiSEM_GAM` are run on the final branches
- what results were shipped
- what upstream synthetic and method-reference files are included

Cannot be fully checked from this repository alone:

- complete retinal raw-image acquisition history
- full raw-image retinal feature extraction pipeline rerun
- a direct APTOS acquisition and preprocessing rerun inside this release

That boundary should be treated as a documented limitation, not hidden.
